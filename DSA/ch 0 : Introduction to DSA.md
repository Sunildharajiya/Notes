# Introduction to Data Structures and Algorithms (DSA)
----------
## What is DSA?

Data Structures and Algorithms (DSA) is one of the most important concepts in computer science and software development. It helps developers write efficient, optimized, and scalable programs.
- Data Structure → A way to organize and store data.
- Algorithm → A step-by-step process to solve a problem.


Together, they help computers process data faster and use memory efficiently

----------
## Why Learn DSA?
Learning DSA improves:
- Problem-solving skills
- Logical thinking
- Coding efficiency
- Interview preparation
- Understanding of software systems

Most tech companies use DSA questions in coding interviews because it shows how well a person can think and optimize solutions.

----------
## Common Data Structures
### 1. Array
Stores elements in a continuous memory location.
Javascript :
```js
const numbers = [1, 2, 3, 4];
```
c++ :
```cpp
const int numbers[] = {1, 2, 3, 4};
```
python :
```py
numbers = [1, 2, 3, 4]
```
### 2. Linked List
A sequence of connected nodes where each node stores data and a pointer.
### 3. Stack
Follows LIFO (Last In, First Out).
Example:
- Undo feature
- Browser history
### 4. Queue
Follows FIFO (First In, First Out).
Example:
- Ticket booking system
- Task schedulin
### 5. Tree
Hierarchical structure used in databases, file systems, etc.
### 6. Graph
Represents connections between nodes.
Example:
- Social networks
- Maps and navigation
### 7. Hash Table
Stores key-value pairs for fast searching.

## Time Complexity - important concept
Time complexity measures how fast an algorithm runs as input size increases.
Common complexities:

| Complexity | Meaning |
| --- | --- |
| O(1) | Constant Time |
| O(log n) | Logarithmic Time |
| O(n) | Linear Time |
| O(n log n)| Efficient Sorting |
| O(n²) | Nested Loops |
